# Resource Library Design

**Date:** 2026-04-05
**Status:** Approved

---

## Context

The homepage has a "Resources" section with a filter + carousel showing 5 hardcoded articles. The article links go to `href="#"` — they don't actually load any content. The section has mobile scaling issues. There is no dedicated resource library page.

The goal is to:
1. Build a proper SEO-driven resource library at `/resources/` — individual article pages that Google can index and rank
2. Fix the homepage section (keep carousel, fix working links, fix mobile, add "View All →")
3. Update article content from 2024 → 2025 deadlines

---

## Architecture

### Hugo Content Structure

```
content/resources/
  _index.md              # Library index page frontmatter
  s-corps.md             # S-Corporation Election guide
  smllc-5472.md          # SMLLC 5472 Election
  form-5471.md           # Form 5471 foreign corp reporting
  due-dates.md           # 2025 Tax Deadlines (updated from 2024)
  1031-exchange.md       # 1031 Exchange Guide
```

Each article frontmatter:
```yaml
---
title: "S-Corporation Election: When It Makes Sense"
description: "Meta description for Google — 150-160 chars"
date: 2025-01-01
practices: [founders, local, realestate]
services: [tax-strategy, tax-compliance]
featured: false
---
```

### New Layouts

```
layouts/resources/
  list.html    # /resources/ library page
  single.html  # /resources/{slug}/ individual article
```

## Pages

### 1. Resource Library — `/resources/`

**Style:** Floating card over fixed gridline background (matches homepage sections — `border-radius: 20px`, heavy `box-shadow`, `margin: 25px`).

**Structure:**
- Green gradient hero (matches service/expertise pages): eyebrow label "Knowledge Base", H1 "Resource Library", subtitle
- Search box (client-side JS — filters cards by title/description text match)
- Filter pills: All Topics · Startups and Online Business · U.S. Expats Abroad · Real Estate Owners · Foreign Business and Investors · International U.S. Citizens · Individuals and Local Business
- Article count label ("Showing N articles")
- 3-column card grid (Hugo `{{ range .Pages }}`)
- Each card: practice tag(s), article title, description, service tag chips, "Read Article →" link to Hugo page

**JS behavior:** Filter pills + search box both filter the visible cards client-side. Hugo renders all cards in the DOM; JS shows/hides based on `data-practices` and `data-services` attributes. No page reload needed.

**Mobile:** 1-column grid below 640px. Search box full-width. Pills wrap.

### 2. Article Page — `/resources/{slug}/`

**Style:** Two separate floating cards over the gridline background (same `border-radius: 16px`, `box-shadow`, `margin: 25px` as homepage sections). Gridlines visible between the two cards.

**Card 1 — Hero (green gradient, `#15482E` → `#3a8b5f`):**
- Breadcrumb: Resources / Article Title
- Practice + service tag pills
- H1 title
- Subtitle/description

**Card 2 — Body (`linear-gradient(90deg, #fff, #edeef7)`):**

Top section — two columns (article ~65%, intake sidebar ~35%):
- **Left:** "← Back to Resource Library" link, markdown content, "Last updated: [date]"
- **Right:** Intake form card (dark green, `align-self: start`) — matches service/expertise page form exactly:
  - H3: "Contact Us" (no subtitle)
  - Label: "Name" → text input
  - Label: "Email Address" → email input
  - Label: "Provide a Brief Background" → textarea
  - "Send Us an Email" button → Formspree `https://formspree.io/f/mkopvwja`
  - Privacy note: "Your information is confidential and protected."
  - Divider
  - "Schedule a Meeting" label + "Book a 30-minute Consultation" sublabel
  - "Open Calendar" button → `https://cal.com/vantis-cpa/30min`

Bottom section — 3-column row (separated by a top border):
- **Related Services** card (white, green left border): links from `services` frontmatter
- **Related Expertise** card: links from `practices` frontmatter using display names from `data/expertise.json` — e.g. "Startups and Online Business", "Individuals and Local Business", "U.S. Expats Abroad" (never slugs or informal names)
- **More Articles** card: auto-generated by Hugo — queries all resource pages sharing a `practices` tag with the current article, excludes current page, shows first 3. Adding a new `.md` file automatically makes it appear in relevant "More Articles" sections with zero extra work.
- **More Articles** card: 2-3 other articles in same practice area

**Mobile:** Hero card full width. Body card: single column (intake form below article, bottom row stacks to 1 column).

### 3. Homepage Section — Updated

**Keep:** Filter dropdown + carousel layout.

**Fix 1 — Working links:** `renderResourceArticles()` currently outputs `href="#"`. Change to `href="/resources/${article.id}/"`.

**Fix 2 — Mobile scaling:** Current: `flex: 0 0 calc(100vw - 60px)` overflows because section has `margin: 25px` on each side. Fix to `calc(min(100vw, 480px) - 48px)` or switch to `width: 100%` on mobile with `flex-shrink: 0`.

**Fix 3 — "View All" link:** Add a "View all articles in the library →" link below the carousel pointing to `/resources/`.

**Fix 4 — 2025 deadlines:** Update the `due-dates` article data to reference 2025 dates.

### 4. Navigation

Add "Resources" as a link in the nav pointing to `/resources/`. The nav already has `<a href="#resources" class="nav-pill">Resources</a>` (homepage anchor). Change this to `/resources/` (the new page). Update mobile menu too.

## Bugs Fixed

1. **"Read Article" links broken** — `href="#"` → `href="/resources/${article.id}/"`
2. **Links break when flipping back and forth** — caused by `href="#"` jumping to top of page on click, re-triggering scroll behavior. Real URLs eliminate this entirely.
3. **Mobile scaling overflow** — card width calculation doesn't account for section's `margin: 25px`. Fixed with corrected calc.
